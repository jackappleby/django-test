  • PIP is like NPM but for Python packages. 
    sudo easy_install pip
  • Install VirtualEnv: 
    sudo pip install virtualenv
    VirtualEnv creates an isolated python directory so you can have multiple Django, PIP package and even Python versions on one machine. Sort of like a, well VIRTUAL ENVIRONMENT.
  • Download Django Test ✔︎
  • Create the virtualenv ✔︎ 
    cd ~/some/path/to/django-test/
virtualenv env
    Note: FFS don't check in your virtualenv, i.e. the 'env' folder. Add 'env' to igtignore
  • ACTIVATE THE VIRTUALENV ✔︎ 
    source env/bin/activate
  • Install the dependancies (this is like "npm install") ✔︎ 
    pip install -r requirements.txt
  • Get the in-built Django development server going (No apache, IIS, blah blah needed!) ✔︎ 
    cd src
python manage.py runserver
  • Go to: 
    http://localhost:8000/
    - This can be changed, IDK know
  • Install the Django Debug Toolbar. Because debugging is cool
  • Check out the admin site
  • Create a Jobs app...
    • django-admin startapp jobs
    • Add the jobs app to the settings
    • Add fields to the Job Model
    • Create your migration 
      python manage.py makemigrations
    • Populate your DB 
      python manage.py migrate
